# 7 天 AI 辅助编程实训 — 总结报告

> 项目：校园个人博客 Campus Blog | 技术栈：Next.js 15 + Flask + Supabase
> 开发工具：VS Code + Claude Code AI 插件 | 部署：Vercel

---

## 一、项目概述

本次实训我独立完成了一个全栈校园个人博客系统。前端基于 Next.js 15 的 App Router 架构，使用 Ant Design 组件库构建了 6 个页面（首页、登录、注册、文章详情、发布/编辑、个人中心），涵盖文章分页列表、全文搜索、分类筛选、热门排行、富文本发布、草稿管理、评论互动、点赞去重、头像上传等功能。后端基于 Python Flask 开发了 11 个 RESTful API 接口，对应用户、文章、评论、点赞四个核心模块，所有接口统一返回 `{code, msg, data}` 格式，并实现了 JWT 鉴权装饰器和全局异常捕获。数据库使用 Supabase（PostgreSQL），设计了 5 张表并配置了 RLS 行级安全策略。项目最终成功部署到 Vercel 平台，前后端全部在线可访问。

## 二、项目难点与解决方案

### 2.1 Supabase JWT 鉴权适配（最大难点）

本地开发时后端登录和 Token 验证始终无法通过，经历了从 HS256 → base64 解码 → 最终发现 Supabase 使用 **ES256 非对称加密算法**的排查过程。解决方案是引入 `PyJWKClient` 从 Supabase JWKS 端点动态获取公钥验证 Token，并安装 `cryptography` 库提供椭圆曲线加密支持。这个过程让我深入理解了 JWT 的对称/非对称加密原理。

### 2.2 Supabase SDK httpx 连接池挂死

项目开发过程中频繁遭遇"第一次请求成功、后续请求全部超时"的问题。经排查发现 Supabase Python SDK 底层使用的 httpx 连接池在 Flask 多进程环境下会挂死。最终将整个数据访问层从 SDK 迁移到 `requests` 库，直接调用 Supabase REST API，每次查询新建连接，彻底解决了卡死问题。

### 2.3 antd v5 / v6 版本兼容

项目初期安装了 antd v6，但 `package.json` 锁定在 `^5.26.0`（v5）。Vercel 构建时 TypeScript 严格检查导致大量 API 不兼容报错。逐一排查并回退到 v5 API。

### 2.4 Vercel Serverless 全栈部署

将 Flask 后端部署到 Vercel 的 Serverless Function 遇到多个坑：Python 入口路径不被识别、只读文件系统报错、API 请求双重前缀、动态路由 404 等。最终通过结构化 `vercel.json` 配置 rewrites、删除 `os.makedirs`、清空 `NEXT_PUBLIC_API_BASE` 等方式逐一解决。

## 三、AI 编程助手使用心得

在本次实训中，我使用了 Claude Code 作为 AI 辅助开发工具，累计提问 16 次以上。AI 在以下几方面发挥了重要作用：

1. **初始代码生成**：一次性生成了 36 个文件的完整项目骨架，极大缩短了从零到一的搭建时间。
2. **报错排查**：面对 Token 验证失败、httpx 连接池挂死、TypeScript 编译报错等复杂问题，AI 能够快速定位根因并给出修复方案。
3. **功能迭代**：草稿箱、头像上传、富文本编辑器完善等功能需求，AI 能直接输出可运行的代码。
4. **部署配置**：Vercel 部署的 vercel.json 配置、Serverless Function 入口等，AI 提供了完整方案。

使用 AI 的心得：AI 是高效的代码生成器和调试助手，但不能盲目信任。每次 AI 输出的代码都需要结合项目上下文验证，特别是版本兼容性、环境差异等细节。好的提问方式（明确技术栈、提供报错日志、说明期望结果）能显著提高 AI 的准确率。

## 四、个人工作与实训收获

本次实训由我个人独立完成全部工作，包括：

- **数据库设计**：5 张数据表的 DDL、索引、触发器、RLS 策略
- **后端开发**：11 个 API 接口，统一鉴权、参数校验、全局异常捕获
- **前端开发**：6 个页面、5 个通用组件、全局状态管理、Axios 封装
- **部署运维**：Vercel 全栈部署、环境变量配置、路由调试

通过这次实训，我深入实践了 Next.js App Router 的服务端组件/客户端组件分离、React Context 全局状态管理、Flask 蓝图路由注册、JWT 鉴权原理、Supabase RLS 行级安全、Vercel Serverless 部署等关键技术点。从一个想法到一套完整可运行的全栈项目，这个闭环的开发体验让我对 Web 全栈开发有了系统性认知，也积累了 AI 辅助编程的实战经验。
